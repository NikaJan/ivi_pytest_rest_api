Перед каждым тестовым классом очищается база данных персонажей:
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/reset
Для удобной работы тестов, чтобы точно знать, в каком состоянии БД


GET characters
1. Запрос на всех персонажей
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/characters
Ожидаемый результат:
Статус код = 200
Не пустой массив персонажей
Произвольный персонаж имеет все требуемые поля


2. Получаем одного персонажа по имени
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/character/3-D%20Man
Ожидаемый результат:
Статус код = 200
Все значения полей персонажа совпадают с заранее известными значениями


3. Получаем несуществующего персонажа
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/character/Ivanov2
Ожидаемый результат:
Статус код = 200
Сообщение = No such name


4. Получаем несуществующего персонажа со спецсимволами в запросе
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/character/%ivanov%
Ожидаемый результат:
Статус код = 400 Bad Request


5. Получаем информацию без отправки имени
curl -u v.milchakova9887@gmail.com:hgJH768Cv23 http://rest.test.ivi.ru/character/
Ожидаемый результат:
Статус код = 404 Not Found
